# المهام المتبقية — Fit Marry
> آخر تحديث: 31 مارس 2026

هذا الملف يلخص **المهام المتبقية الفعلية حالياً** بعد آخر دفعة تطوير على مسار المطابقة الجادة، تبادل التواصل، الولي، وقصص النجاح.

---

## 0. ما اكتمل في هذه الدفعة

- دعم حالات `رفض / إلغاء / انتهاء صلاحية` لطلب تبادل التواصل في الباك-إند.
- إضافة endpoints خاصة بالرفض والإلغاء.
- تحديث الشات لدعم:
  - الموافقة
  - الرفض
  - الإلغاء
  - إعادة الطلب بعد الرفض/الإلغاء/الانتهاء
- تحديث شاشة المطابقات لتقوم بـ:
  - ترتيب الحالات الواردة أولاً
  - إبراز المحادثات العاجلة
  - إضافة قسم `يتطلب موافقتك الآن`
- تحديث الإشعارات وجرس الإشعارات لدعم أولويات طلبات تبادل التواصل.
- إضافة اختبارات backend مركزة لـ `CompatibleMatchService`.
- إضافة مؤشرات المسار الجاد إلى Dashboard في الأدمن.

> ملاحظة: إنشاء Prisma migration جديد لهذه الدفعة **لم ينجح بعد** بسبب عطب قديم في سجل migrations الحالي، وليس بسبب التغييرات الجديدة نفسها.

## 1. أعلى أولوية الآن

### 1.1 إصلاح سجل Prisma migrations
- إصلاح migration القديمة `20260209111715_update_schema_new_rules` التي تمنع إنشاء migration جديدة على shadow DB.
- بعد الإصلاح: إنشاء migration رسمية لمسار `contact exchange state machine`.

### 1.2 إكمال اختبارات المسار الجاد
- إضافة اختبارات للحالات السلبية المتبقية:
  - لا يمكن للمرسل رفض طلبه نفسه.
  - لا يمكن لغير المرسل إلغاء الطلب.
  - لا يمكن إضافة وسيلة التواصل قبل حالة `APPROVED`.
  - لا يمكن كشف بيانات الولي الحساسة من الـ public payloads.

---

## 2. مهام تجربة المستخدم في المسار الجاد

### 2.1 تحسين شاشة المطابقات
- إضافة فلاتر أدق للحالات:
  - بانتظارك
  - بانتظار الطرف الآخر
  - جاهز لتبادل التواصل
  - مكتمل
- إضافة إجراءات سريعة من البطاقة نفسها دون الحاجة لفتح الشات في كل مرة.

### 2.2 تحسين شاشة الشات للمطابقات الجادة
- إضافة timeline مبسط يوضح المرحلة الحالية:
  - إعجاب
  - توافق متبادل
  - طلب تبادل تواصل
  - موافقة
  - إضافة وسيلة تواصل
  - مكتمل
- جعل حالة الولي/الوصي أكثر وضوحاً عند توفرها دون كشف بيانات حساسة.
- تحسين نسخ الرسائل التوضيحية بحيث يفهم المستخدم المرحلة التالية مباشرة.

### 2.3 تحسين شاشة قصص النجاح
- إضافة إنشاء قصة نجاح من داخل التطبيق للمستخدمين المؤهلين فقط.
- إضافة ربط أقوى بين شاشة الإنجاز النهائي `match-success` ونشر القصة لاحقاً.

## 3. مهام الباك-إند المتبقية

### 3.1 تحليلات وتشغيل المنتج
- إضافة counters أو analytics للمراحل التالية:
  - عدد التوافقات المتبادلة
  - عدد طلبات تبادل التواصل المرسلة
  - نسبة الموافقة
  - نسبة الإكمال حتى تبادل التواصل النهائي
- تجهيز هذه البيانات للاستخدام لاحقاً في لوحة التحكم.

### 3.2 مراجعة الأمان والخصوصية
- التأكد أن بيانات الولي الحساسة لا تخرج في أي response عام أو غير مقصود.
- مراجعة كل payloads الخاصة بالمحادثات والملفات العامة بعد التعديلات الأخيرة.

## 4. مهام لوحة التحكم

### 4.1 دعم ميزات المسار الجاد داخل الأدمن
- صفحة لمتابعة مؤشرات المطابقة الجادة وتبادل التواصل.

### 4.2 توسيع الأدمن بعد اكتمال الصفحات الأساسية
- إضافة مؤشرات تشغيلية وتجارية أعمق في الـ dashboard.
- إضافة عرض أوضح لتحولات المسار الجاد عبر الزمن، لا كأرقام لحظية فقط.

## 5. مهام جودة واختبار

### 5.1 اختبارات وظيفية
- إضافة اختبارات للحالات السلبية:
  - لا يمكن إضافة التواصل قبل الموافقة المتبادلة
  - لا يمكن الوصول لبيانات الولي الحساسة من الـ public profile

### 5.2 اختبارات واجهة وتجربة
- تجربة المسار الكامل على الموبايل بحسابين حقيقيين من البداية للنهاية.
- التأكد من تزامن socket في الحالات التالية:
  - وصول طلب جديد
  - موافقة الطلب
  - إضافة وسيلة التواصل
  - اكتمال الحالة

### 5.3 تنظيف التقارير والتوثيق
- تحديث الخطة الرئيسية أو التقرير النهائي لإبراز مسار الزواج الجاد الجديد.
- معالجة إشارات TypeScript/diagnostics المتقلبة في ملفات i18n إن استمرت.

## 7. ملاحظات

- المسار الأساسي الحالي يعمل: توافق متبادل → طلب تبادل تواصل → موافقة → إضافة وسيلة تواصل → اكتمال.
- المتبقي الآن ليس تأسيسياً بقدر ما هو **إكمال وتشغيل ورفع جودة المنتج**.
- أعلى عائق هندسي مباشر الآن هو مشكلة Prisma migration القديمة التي تمنع إخراج migration جديدة بشكل نظيف.
