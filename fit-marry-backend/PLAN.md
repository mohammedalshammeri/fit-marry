# خطة تنفيذ الباك-إند (Fit-Marry)

## الهدف
بناء باك-إند إنتاجي لمنصة توافق/زواج يدعم تطبيقات الجوال ولوحة تحكم الإدارة عبر نفس الـ API، مع تركيز قوي على الأمان، الـ RBAC، سجلات التدقيق، الـ OTP، المحافظ والدفع بالدقائق، وضوابط المحادثات.

---

## خارطة الطريق (مقسّمة حسب المطلوب)

### A) Prisma Schema + Migrations + Seed
- تصميم مخطط قاعدة البيانات بالكامل (جداول، علاقات، فهارس، قيود).
- إنشاء أول Migration.
- Seed للأدوار والصلاحيات والإعدادات الأساسية.
- **الحالة:** ✅ تم تنفيذها في هذا المجلد.

### B) Auth + OTP + Device Restriction
- تسجيل/تسجيل دخول عبر OTP (بريد/هاتف).
- منع أكثر من حساب لكل جهاز (تخزين `deviceIdHash`).
- JWT Access/Refresh مع تدوير الـ Refresh Token.
- حماية المعدّل (Rate Limiting) لمسارات الحساسة.

### C) Profiles + Discovery
- CRUD للملف الشخصي.
- منطق إخفاء/إظهار المستخدمين حسب القواعد (نشاط/قيود/تطابق).
- عوامل تصفية التوافق الأساسية.

### D) Likes + Conversation Creation Rules
- نظام الإعجاب والاستقبال والحد الأعلى.
- إنشاء محادثة تلقائيًا عند التطابق.
- منع المحادثات المتعددة لنفس المستخدم.

### E) Wallet + Transactions
- محفظة بالدقائق/الأرصدة.
- التسجيل/تعديل الملف مقابل رسوم.
- خصم دقائق الدردشة والمكالمات.

### F) Messages + Media Rules
- رسائل نصية/صور.
- منع الصور أول 7 أيام.
- View-Once مع تتبع المشاهدات.
- تكامل Cloudinary عبر توقيع موقّت.

### G) Complaints + Admin Limited Chat Access
- نظام الشكاوى وسير العمل.
- صلاحية وصول محدود للمحادثات عند وجود شكوى.

### H) Admin APIs + Audit Logs + Settings + Banners
- إدارة المستخدمين والمشرفين والصلاحيات.
- سجلات تدقيق كاملة.
- إدارة الإعدادات والبنرات.

### I) Notifications + Tests
- جداول التنبيهات وواجهة طابور غير متزامنة.
- اختبارات وحدات وE2E للأجزاء الحرجة.
